HISTORIA DE JavaScript

JavaScript es un "lenguaje" de programacion para HTML y la web
es facil de aprender y utilizar, ya que nos permite manipular etiquetas HTML y hacer bastantes funciónalidades, tando para el diseño del front-end CSS como para el desarrollo de apis y procesos del back-end

JavaScript fue desarrollado originalmente por Brendan Eich de Netscape con el nombre de Mocha, el cual fue renombrado posteriormente a LiveScript, para finalmente quedar como JavaScript

fue creado en 10 dia, con el fin de reemplazar lo applets de los lenguajes como java y c#, y asi poder generar paginas web mas dinamicas
esta basado en el estandar de ECMAScript 6 que es la ultima version de los estandares para lenguajes de programacion basados en scripts, como lo son TypeScript de microsoft, y JavaScript

ECMA es una organizacion internacinal parecida a icontec, que se encarga de tener unas normas estandarizadas de los lenguajes de programacion, en otras palabras, ECMA es la organizacion encargada de certificar de estandar del uso de los lenguajes de programacion, como su sintaxix, palabras reservadas, y estructura

aunque JavaScript esta diseñado para ser visualizado por un navegador, e interpretado por un motor del mismo navegador, JavaScript tambien puede ser usado en entrornos de sistema operativo, corriendo sobre un motor en el sistema operativo, utilizando el Framework NODE.JS

para poder usar JavaScript en paginas web, es necesario que este codigo sea interpretado por un motor, que varia segun el navegador que estemos usando, ya que este motor, se basa tambien en la version de ECMAScript sobre el cual este desarrollado

ya habiendo hablado un poco de donde viene JavaScript, ¿es realmente un lenguaje de programacion?


si comparamos JavaScript con un lenguaje de programacion, como Java, C++ o C# notamosq ue hay muchas cosas que los demas lenguajes poseen, pero en JavaScript estan ausente, como lo son el caso de las clases, metodos, tipado de variables entre otras.

tomando en cuenta que JavaScript conserva el mismo principio de un lenguaje de programacion, ya que usa instrucciones, y se comunica mediante ellas con una, interface, pagina o aplicacion, este no cumple con todos los que estandariza a los lenguajes anteriormente mencionados, por lo que se podria decir que JavaScript es un dialecto o adaptacion de el lenguaje estandarizado por las ECMAScript lo cual no lo hace 100% un lenguaje de programacion, y aunque mantenga su funciónalidad, no llega a ser complertamente u lenguaje de programacion completo.
